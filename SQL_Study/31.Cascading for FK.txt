1} On DELETE Cascading :-
			  When we create foreign key using this option , it deletes referencing rows in the child table
			  when referenced row is deleted in parent table which has primary key.

2} On UPDATE Cascading :- 
			  When we create foreign key using UPDATE CASCADE the referencing row are updated in child table
			  when reference row is updated in parent table which has primary key

	CREATE TABLE student (
		id INT PRIMARY KEY, 
		coursed INT,
		FOREIGN KEY(courseID) REFERENCES course(id)
		ON DELETE CASCADE
		ON UPDATE CASCADE
			);
